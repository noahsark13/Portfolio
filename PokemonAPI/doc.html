DOCUMENTATION PROCESS

Started off by setting up the basic UI structure using html and css. I decided a grid layout for the main portion would work well, representing a design similar to the traditional pokedex in the pokemon games.
Started on functionality by using some left over javascript from the giphy assignment. After looking back over it, I've come to understand the basics of API requesting, and their asynchronous functionality.
Using previous coding knowledge, I looked up how to code certain functionalities using stackoverflow and other javascript resources. Using this, I built the various different chunks of my page, using continuous API gets. Ran into several bugs since you can't return specific data from httprequest .onload functions. Would have to use callback functions to achieve a somewhat expected result.
Continued to same process for various parts of my page, getting different api data, then constructing new html elements with said information.
Found issues again when generating the display for all current pokemon. Building a long string every load was far too intensive on the page, causing it to have to load for a solid 5 seconds, before everything became function. To get around this, I built individual divs and appended them, resulting in a much faster load speed.
Then added visual design to the layout, finishing up the project.
